This site documents [`@openmrs/ngx-formentry`](https://github.com/openmrs/openmrs-ngx-formentry), the optional **Angular Form Engine** for OpenMRS 3 (O3). It also retains clearly labelled documentation for the archived [AMPATH Form Builder](https://github.com/ampath/ngx-openmrs-formbuilder) for teams maintaining an existing deployment.

The OpenMRS Reference Application uses the **React Form Engine** by default, and its built-in O3 Form Builder creates and previews forms with that React engine. It does not preview Angular Form Engine behavior. There is currently no supported visual builder for the Angular engine. Although both engines consume the shared [O3 form schema](https://github.com/openmrs/openmrs-contrib-json-schemas/blob/main/form.schema.json), schema conformance does not guarantee that every field or feature behaves the same in both engines.

If you use the default Reference Application workflow, start with the [canonical O3 Forms guide](https://o3-docs.openmrs.org/en-US/docs/forms-in-o3/). If your distribution explicitly includes the Angular engine, read [About this documentation](/docs/about-this-documentation) before using the reference pages on this site.
